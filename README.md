# 1-WavTube

制作期間 : 約1年

WavTubeは、YouTubeの動画を高音質な音声ファイル(WAV / MP3)としてダウンロードし、同時に楽曲のBPMとキー(調)を自動解析してファイル名に付与するデスクトップアプリケーションです。
クリエイターやDJ、トラックメイカーがリファレンス楽曲やサンプリング素材を収集する際の過程を大幅に効率化します。

## 技術的な工夫点

本アプリケーションは、単なるダウンロードツールではなく、解析の正確性とアプリケーションとしての堅牢性にこだわって開発しました。

* **信頼性の高いダウンロードエンジンの選定 (`yt-dlp`)**
    頻繁な仕様変更が起こるYouTubeのスクレイピングにおいて、保守性が高くダウンロード速度に優れる `yt-dlp` を採用しました。これにより、安定した動作を実現しています。
  
* **音楽理論に基づいた本格的なキー解析 (`librosa` + Krumhansl-Schmucklerアルゴリズム)**
    音声の調判定には、単純なピーク検出ではなく、`librosa.feature.chroma_cqt` を用いてクロマ特徴量を抽出し、音楽心理学で実績のある「Krumhansl-Schmucklerキーテンプレート」との内積を計算する独自ロジックを実装しました。これにより、精度の高い長調(Major)/ 短調(Minor)の判定を可能にしています。
  
* **配布を前提としたパス解決ロジックの実装**
    PyInstaller等で単一の実行ファイル(.exe)化した際にも `ffmpeg` やアイコン画像が正しく読み込まれるよう、`sys._MEIPASS` を考慮した一時ディレクトリのパス解決関数(`resource_path`)を実装しています。

## ユーザー観点での工夫

「音楽素材を集めるユーザー」の思考プロセスを分析し、クリック数と認知負荷を最小限に抑えるUI/UXを設計しました。

* **ファイル名へのBPM/キー自動付与**
    ダウンロードした楽曲をDJソフトやDAWにインポートする際、事前にBPMやキーが分かっていると作業がスムーズです、本ツールでは解析結果を `ファイル名 - 120BPM C Maj.wav` のように自動付与するため、ユーザーが別の解析ソフトを立ち上げる手間を省きました。
  
* **クリップボードからのワンクリック貼り付け**
    ブラウザでURLをコピーした後、入力欄を右クリックして貼り付ける手間を省くため、専用の「貼り付け」ボタンを配置しました。
  
* **OS標準パスへの設定自動保存**
    ユーザーが毎回保存先フォルダを指定し直さなくて済むよう、Windows (`%LOCALAPPDATA%`) と macOS/Linux (`~/.config`) の各OSの標準的な設定ファイルディレクトリを自動判別し、保存先パスを永続化する機能を実装しています。再度exeを起動したとき前回の保存先をファイルから参照しています(邪魔にならないよう、隠しファイルで保存します)

## 課題に対するアプローチ

* **GUIのフリーズ問題と非同期処理による解決**
    音声のダウンロードからBPM/キーの解析(特にCQT変換)にかけては、非常に重い処理となります。当初、メインスレッドで実行すると解析中にUIが「応答なし」になってしまう課題がありました。そこで `threading` モジュールを用いてダウンロードと解析処理をバックグラウンドスレッドに分離し、UIのプログレスバーやステータス表示をリアルタイムに更新できるように改善しました。
  
* **一時ファイルの確実なクリーンアップ**
    ダウンロードから変換の過程で作成される一時ファイルがストレージを圧迫しないよう、`tempfile.mkdtemp()` で専用の一時ディレクトリを作成し、処理の成否に関わらず `finally` ブロックで確実に `shutil.rmtree` で削除する堅牢なエラーハンドリングを実装しました。
